Overview

Executive Summary: This project focuses on testing a React Native To-Do Application across Android and iOS devices. Main areas include task creation, editing, reminders, and cloud sync with Firebase. Basic Appium automation is implemented for Android; iOS remains mostly manual.
Current Status: In Progress — Additional performance checks for large data sets (1000+ tasks) planned for <>.

This project involves testing a Mobile To-Do Application that allows users to create, edit, and delete tasks, set reminders, and sync tasks across multiple devices. The primary goal is to ensure seamless user experience, proper data handling, and stable performance on Android and iOS.

Test Environment

Devices & Platforms:

  • Android 11 (Samsung Galaxy S10) real device
  • Android Emulator (Pixel 3, Android 12)
  • iPhone 12, iOS 15 (real device)
  • iOS Simulator (Xcode 13)

Tools & Setup:

  • ADB / Xcode CLI for installing and debugging apps
  • Postman for backend API testing (sync endpoints)
  • SQLite DB (on-device) and Firebase (cloud sync)
  • JIRA for bug tracking

The app is built in React Native, connecting to a Node.js + Firebase backend for data persistence and push notifications.

Test Cases
TC-001: Create a New Task (Positive)

Steps:

  1. Open the To-Do app.
  2. Tap the "Add Task" button.
  3. Enter a title (e.g., "Grocery Shopping") and optional description.
  4. Set a due date and tap "Save".

Expected: The new task appears in the list with correct title/date.

Status: Pass

TC-002: Create Task with Empty Title (Negative)

Steps:

  1. Open the To-Do app.
  2. Tap "Add Task".
  3. Leave title blank, enter optional description.
  4. Try to save the task.

Expected: An error message "Title is required" is displayed. Task is not created.

Status: Pass

TC-003: Edit Existing Task

Steps:

  1. Select an existing task from the list.
  2. Tap "Edit". Change the title, date, or description.
  3. Tap "Save".

Expected: Task updates with the new data, and changes persist after app reload.

Status: Pass

TC-004: Mark Task as Completed

Steps:

  1. Tap the checkbox icon next to a task.
  2. Confirm that the task moves to the "Completed" section or changes appearance.

Expected: Task is visually indicated as completed; remains in completed state upon refresh.

Status: Pass

API Testing

The app syncs tasks with a Node.js + Firebase backend. We tested the /sync and /tasks endpoints via Postman:

POST /api/tasks (Add new task)

Request: JSON body with title, description, dueDate.

Expected: 201 Created, returns JSON with taskId and createdAt.

Result: Pass

GET /api/tasks (Get user tasks)

Expected: 200 OK, an array of tasks for the current user.

Negative Test: If invalid token, return 401 Unauthorized.

Result: Pass

DB Testing

Local DB (SQLite): For offline mode, tasks are stored locally in an SQLite database. On app launch, we verify the existence of the tasks table with correct schema.

Check local tasks table after adding a new task

Query: SELECT title, due_date FROM tasks WHERE title='Grocery Shopping';

Expected: One row with the correct title, due_date.

Result: Pass

Cloud DB (Firebase): Verified that after successful sync, tasks appear in the user’s Firebase collection.

Regression

After each new feature (e.g., push notifications, tag-based categorization), we run a smoke test on critical flows (Create/Edit tasks, Sync).

A partial regression covers functionalities closely linked to the updated module (e.g., changes to "Completed tasks" triggers re-testing of marking tasks done, re-testing sync behavior, etc.).

Automation

We implemented a basic Appium suite for Android to automate crucial test flows (create task, edit task, mark completed). For iOS, manual testing was performed due to limited device availability.

CI/CD Integration:

  • GitHub Actions triggers the Appium suite on each pull request (Android Emulator).
  • Postman tests for the API run via Newman in the same pipeline.

This setup ensures that critical regressions are caught before merging code changes.

Bug Reports
BUG-001: App Crash on Editing Task Offline

Status: Open

Priority: High

Description: When device is offline, editing an existing task leads to an immediate app crash on some Android devices.

Steps to Reproduce:

  1. Disable Wi-Fi/data on the device.
  2. Open the app and select an existing task.
  3. Tap "Edit", change the title.
  4. Tap "Save" & observe the crash.

Environment: Android 11, v1.2.0 of the app.

Expected Result: Changes are saved locally and later synced when online, no crash.

Attachments: Logcat Output

BUG-002: Missing Reminder Notification iOS

Status: Open

Priority: Medium

Description: Reminders do not appear for tasks on iOS if the app is not in foreground.

Steps to Reproduce:

  1. Open the app on iPhone 12, create a task with a reminder time ~2 min in future.
  2. Close the app (swipe up) & wait.
  3. No notification is received at the specified reminder time.

Environment: iOS 15, v1.2.0 of the app.

Expected Result: Local or push notification should appear even if the app is in the background.

Attachments: Screenshot of Notification Settings

Summary

Overall, the mobile To-Do app meets most functional requirements, with tasks smoothly created, edited, and synced across devices. The offline crash (BUG-001) and iOS reminders issue (BUG-002) remain open but are being addressed in the next sprint.

Future improvements include refining push notification handling on iOS and implementing automated iOS tests with Appium or XCUITest. Also, performance under large data sets (1000+ tasks) will be evaluated in an upcoming release.
